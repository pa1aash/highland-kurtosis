## 2. Geant4 Source Code Inventory

### MCSHighland.cc (main)
- **Lines:** 53
- **Purpose:** Entry point. Creates RunManager, initializes DetectorConstruction, PhysicsList, ActionInitialization. Supports interactive mode (no args) or batch mode (macro file as arg1, optional seed as arg2).
- **Random engine:** CLHEP::RanecuEngine, seeded from `time(nullptr)` or argv[2]

### DetectorConstruction.cc / .hh
- **Lines:** ~844 (largest file)
- **Class:** `DetectorConstruction` (note: typo in name, missing 'o' — "Construction")
- **Geometries implemented (7):**
  1. `kSolidPLA` — solid PLA block, volume `SolidPLA`
  2. `kAirOnly` — no target placed (null test)
  3. `kRectilinear` — CSG walls: `XWall` + `YWall` slabs in air `Target` volume
  4. `kHoneycomb` — voxelised 2D hex pattern, volume `HCWall`, pixel=0.1mm, extruded along z
  5. `kGyroid` — voxelised 3D TPMS via `|sin(kx)cos(ky)+sin(ky)cos(kz)+sin(kz)cos(kx)| < threshold`, volume `GVoxPLA`. Also supports STL import via CADMesh (`Gyroid` volume)
  6. `kCubic` — CSG walls in x/y/z: `CubXWall`, `CubYWall`, `CubZWall`
  7. `kVoronoi` — voxelised 3D CVT (centroidal Voronoi tessellation), volume `VVoxPLA`. Also supports STL import via CADMesh (`Voronoi` volume)
- **Materials defined:**
  - `PLA` — C3H4O2, density 1.24 g/cm3 (X0 ≈ 315 mm)
  - `G4_AIR` — from NIST database
  - **No silicon or tungsten defined**
- **Key configurable parameters:**

| Parameter | Member | Default | Messenger command |
|-----------|--------|---------|-------------------|
| Geometry type | `fGeometryType` | `kSolidPLA` | `/MCS/det/geometry` |
| Infill percent | `fInfillPercent` | 100.0 | `/MCS/det/infill` |
| Cell size | `fCellSize` | 4.0 mm | `/MCS/det/cellSize` |
| Wall thickness | `fWallThickness` | 0.4 mm | `/MCS/det/wallThickness` |
| Sample thickness (z) | `fSampleThickness` | 10.0 mm | `/MCS/det/sampleThickness` |
| Sample width (xy) | `fSampleWidth` | 20.0 mm | `/MCS/det/sampleWidth` |
| STL file path | `fSTLFile` | "" | `/MCS/det/stlFile` |

- **Target region:** 0.1 mm production cut (vs 1.0 mm global)
- **Step limits:** 0.1 mm `G4UserLimits` on all PLA volumes (0.5 mm on honeycomb voxels)
- **Infill tuning:** Honeycomb, gyroid, cubic, and voronoi all use bisection search to tune geometry parameters (cell spacing or threshold) to match requested infill percentage. Honeycomb and cubic use Monte Carlo sampling with beamSigma=5mm; gyroid uses uniform sampling.
- **Slab thickness:** `fSampleThickness` controls overall target thickness and is independent of lattice geometry — **solid PLA slab thickness CAN be changed via `/MCS/det/sampleThickness`**.
- **Note:** Cubic lattice constructor (lines 533-681) is correct: `yBounds` is initialized with `{-halfW}` at line 645, and `ix` is a proper loop variable in the `for` loop at line 657. Walls are placed in all three orientations (X, Y, Z) with proper bounds-based clipping to avoid overlaps.

### DetectorMessenger.cc / .hh
- **Class:** `DetectorMessenger`
- **7 commands exposed:** `/MCS/det/geometry`, `/MCS/det/infill`, `/MCS/det/cellSize`, `/MCS/det/wallThickness`, `/MCS/det/sampleThickness`, `/MCS/det/sampleWidth`, `/MCS/det/stlFile`
- All available in `G4State_PreInit` and `G4State_Idle`

### PhysicsList.cc / .hh
- **Class:** `PhysicsList`
- **EM physics:** `G4EmStandardPhysics_option4` (EMZ) — **hardcoded**, no command to switch
- **Other physics:** G4EmExtraPhysics, G4DecayPhysics, G4HadronElasticPhysics, G4HadronPhysicsFTFP_BERT, G4StoppingPhysics, G4IonPhysics, G4StepLimiterPhysics
- **MSC parameters (ConstructProcess):**
  - `MscRangeFactor` = `fRangeFactor` (default 0.04)
  - `MscSkin` = 3
  - `MscStepLimitType` = `fUseSafetyPlus`
  - `MscThetaLimit` = 0.2
- **Production cuts:** 1.0 mm global (0.1 mm in Target region set by DetectorConstruction)
- **Messenger commands:**

| Command | Type | Default |
|---------|------|---------|
| `/MCS/phys/rangeFactor` | double [0,1) | 0.04 |
| `/MCS/phys/mscStepMax` | double+unit | 0.1 mm |

### PrimaryGeneratorAction.cc / .hh
- **Class:** `PrimaryGeneratorAction`
- **Particle:** `e-` (electron) — **hardcoded via `FindParticle("e-")`**, no command to switch
- **Default energy:** 4.0 GeV (changeable via standard `/gun/energy`)
- **Beam direction:** (0, 0, 1) — along +z
- **Starting position:** (x, y, -50 mm) with Gaussian or pencil beam
- **Beam spot:** Gaussian sigma = 5.0 mm (default), or pencil (x=y=0)
- **Messenger commands:**

| Command | Type | Default |
|---------|------|---------|
| `/MCS/gun/pencilBeam` | bool | false |
| `/MCS/gun/beamSigma` | double+unit | 5 mm |

- **Note:** Energy is set via Geant4's built-in `/gun/energy`, NOT `/MCS/gun/energy`

### SteppingAction.cc / .hh
- **Class:** `SteppingAction`
- **Tracks only primary particle** (TrackID == 1)
- **Volume detection:** Uses lambda `isTargetVolume()` to check if volume name is one of: `Target`, `SolidPLA`, `PLAFill`, `XWall`, `YWall`, `HCWall`, `GVoxPLA`, `VVoxPLA`, `Gyroid`, `Voronoi`, `CubZWall`, `CubXWall`, `CubYWall`
- **PLA path accumulation:** `isPLAVolume()` — same list minus `Target` (which is air)
- **Entry detection:** first transition from non-target to target volume
- **Exit detection:** transition from target to `World` volume
- **Records:** entry position, entry momentum direction, exit momentum direction, exit energy, cumulative PLA path length

### EventAction.cc / .hh
- **Class:** `EventAction`
- **Ntuple columns filled (7):**

| Column index | Name | Unit | Description |
|-------------|------|------|-------------|
| 0 | `theta_x` | rad | Projected scattering angle x: `exit_tx - entry_tx` where `tx = px/pz` |
| 1 | `theta_y` | rad | Projected scattering angle y: `exit_ty - entry_ty` |
| 2 | `theta_space` | rad | Space angle: `sqrt(theta_x² + theta_y²)` |
| 3 | `energy_out` | GeV | Exit kinetic energy |
| 4 | `entry_x` | mm | Entry x position |
| 5 | `entry_y` | mm | Entry y position |
| 6 | `pla_path` | mm | Total PLA path length traversed |

- Only fills ntuple row if `fHasExit == true` (particle exited target)

### RunAction.cc / .hh
- **Class:** `RunAction`
- **Ntuple name:** `scattering`
- **Output format:** ROOT (`.root`) via G4AnalysisManager with ntuple merging enabled
- **Default filename:** `MCSOutput` (produces `MCSOutput.root`)
- **Messenger command:** `/MCS/output/fileName` — sets output filename (without extension)

### ActionInitialization.cc / .hh
- **Class:** `ActionInitialization`
- Wires together PrimaryGeneratorAction, RunAction, EventAction, SteppingAction

## 4. Analysis Pipeline

### analyze_mcs.py (analysis/)
- **Purpose:** Main analysis pipeline — loads ROOT simulation output, computes scattering statistics, fits distributions, produces publication figures.
- **Input:** `--input-dir` (directory of .root files) or `--single-file`; optional `--sweep0` (sweep0_summary.json for universality plot)
- **Output:** `results/analysis_results.json`, `results/fig1_kurtosis_vs_infill.png`, `results/fig2_universality.png`, `results/map_*.png` (position-resolved maps)
- **Key functions:**
  - `highland_sigma_rad(x_over_X0, p_gev)` — Highland formula: `(13.6e-3/p) * sqrt(x/X0) * (1 + 0.038*ln(x/X0))`
  - `excess_kurtosis(data)` — **`scipy.stats.kurtosis(fisher=True, bias=False)`** — unbiased Fisher excess kurtosis estimator
  - `rms_width(data)` — `np.std(data, ddof=1)`
  - `core_gaussian_width(data, central_fraction=0.98)` — std of central 98% of data
  - `tail_fraction(data, n_sigma=3)` — fraction of data beyond 3σ
  - `fit_two_gaussian(data, n_bins=200)` — two-Gaussian mixture fit via `scipy.optimize.curve_fit`
  - `apply_cuts(data, beam_energy_gev, ...)` — applies energy, fiducial, and angle cuts; prints per-cut removal counts
  - `analyze_config(data, ...)` — main per-file analysis
  - `position_resolved_analysis(data, bin_size_mm=1.0)` — 2D map of σ and κ vs entry position
- **Quality cuts:** Configurable via command-line arguments (applied before `analyze_config`):
  - `--energy-cut-fraction` (default 0.9): keep events with `energy_out > fraction * beam_energy`
  - `--angle-cut-sigma` (default 10): keep events with `|theta_x| < N * sigma_Highland_solid`
  - `--fiducial-x` (default 5.0 mm): keep events with `|entry_x| < value`
  - `--fiducial-y` (default 10.0 mm): keep events with `|entry_y| < value`
  - `--beam-energy` (required when cuts applied): beam energy in GeV for Highland sigma calculation
  - `--no-cuts`: skip all cuts (backward compatible with previous behavior)
  - Highland sigma for angle cut uses solid PLA: `highland_sigma_rad(10.0/315.0, beam_energy)`
- **Constants:** `PLA_X0_MM = 315.0`, `PLA_X0_CM = 31.5`
- **Kurtosis error:** `sqrt(24/N)` (Gaussian approximation)

### proposal_analysis.py (analysis/)
- **Purpose:** Stricter analysis for proposal — applies energy, angle, and fiducial cuts.
- **Input:** ROOT files from `--data-dir` (default `data/proposal/`)
- **Output:** `--output-dir` (default `results/proposal/`) containing `proposal_summary.json`
- **Quality cuts (configurable via CLI):**
  - `--energy-cut-fraction` (default 0.9): `energy_out > fraction * p_GeV`
  - `--angle-cut-sigma` (default 10): `|theta_x| < N * sigma_Highland` (Nσ clip)
  - `--fiducial-x` (default 5.0 mm): `|entry_x| < value`
  - `--fiducial-y` (default 10.0 mm): `|entry_y| < value`
- **Kurtosis:** `scipy.stats.kurtosis(fisher=True, bias=False)` with 1000-resample bootstrap (seed=42)
- **Constants (now defaults):** `PLA_X0_MM = 315.0`, `SAMPLE_T_MM = 10.0`, `n_cut_sigma_DEFAULT = 10`, `FIDUCIAL_X_MM_DEFAULT = 5.0`, `FIDUCIAL_Y_MM_DEFAULT = 10.0`, `ENERGY_CUT_FRACTION_DEFAULT = 0.9`

### universal_equation_final.py (analysis/)
- **Purpose:** Validates universal kurtosis equation `κ = (3+κ_M)/f - 3` against Geant4 data.
- **Input:** ROOT files from `data/proposal/`, `data/sweep0/sweep0_summary.json`
- **Output:** Multi-table analysis (console)
- **Quality cuts:** Energy cut `>0.9*p`, angle cut `10*σ_Highland`, fiducial `|x|<5, |y|<10`
- **Bootstrap:** 2000 resamples, seed=42
- **PLA hit threshold:** `pla_path > 0.1 mm`

## 6. Build System

### CMakeLists.txt
- **CMake:** 3.16-3.27
- **Project:** `MCSHighland` v1.0.0, C++ only
- **Geant4:** Required, with optional `ui_all vis_all` (ON by default)
- **CADMesh:** Optional (`-DWITH_CADMESH=ON`), adds `-DUSE_CADMESH` define for STL import of gyroid/voronoi geometries. Falls back to voxelised construction if not found.
- **Source collection:** `file(GLOB sources src/*.cc)`, `file(GLOB headers include/*.hh)`
- **Executable:** `MCSHighland` from `MCSHighland.cc` + all src/include
- **Macros copied:** `macros/*.mac` → build directory
- **STL copied:** `geometry/*.stl` → build directory (but `geometry/` doesn't exist yet)
- **Install:** `MCSHighland` to `bin/`

### install_geant4.sh (scripts/)
- **Conda method (recommended):** Creates `g4highland` conda env with: geant4, root, cmake, compilers, python=3.12, numpy, scipy, matplotlib, scikit-image, uproot, awkward
- **Source method:** Downloads Geant4 11.2.2 from GitHub, builds with cmake, installs to `~/geant4/install`
- **Check mode:** `--check` verifies geant4-config, ROOT, Python packages
- **Python deps (source method):** numpy, scipy, matplotlib, scikit-image, numpy-stl, uproot, awkward

### run_all_sweeps.sh (scripts/)
- **Pipeline:**
  1. Build (`cmake .. && make`)
  2. Generate macros (`python3 scripts/generate_macros.py`)
  3. Run sweeps 0-6 + controls
  4. Run analysis (`python3 analysis/analyze_mcs.py`)
- **Options:** `--build-only`, `--sweep N` (run single sweep), `--jobs N` (parallel)
- **Sweep 0:** Ray-trace (no Geant4) via `scripts/ray_trace_sweep0.py`
- **Sweeps 1-6:** Run generated macros from `macros/generated/`
- **Controls:** `run_solid_control.mac`, `run_air_control.mac`
- **Output handling:** Moves `.root` and `.csv` files from build/ to data/

### run_proposal_campaign.sh (scripts/)
- **27 configurations:** 4 controls + 4×4 lattice infills + 2×2 energy scans
- **50k events per config** (except Voronoi: 200k)
- **Generates per-run macro files** in `data/proposal/`
- **Seed:** Deterministic from config name via `cksum`

## 7. Gaps and Missing Capabilities

### Gap 1: Run solid PLA slabs at arbitrary thicknesses independent of lattice mode
- **Status: ALREADY SUPPORTED**
- `/MCS/det/sampleThickness` works with `/MCS/det/geometry solid` to set arbitrary slab thickness (0.1-10+ mm)
- The parameter is independent of lattice geometry

### Gap 2: Switch material from PLA to silicon or tungsten
- **Status: IMPLEMENTED**
- **Files:** `src/DetectorConstruction.cc`, `src/DetectorMessenger.cc`, `include/DetectorConstruction.hh`, `include/DetectorMessenger.hh`
- **Details:** Added `fSilicon` (`G4_Si`, 2.33 g/cm3, X0=93.7 mm), `fTungsten` (`G4_W`, 19.3 g/cm3, X0=3.5 mm) via NIST manager. `fTargetMaterial` pointer selects active material (default PLA). `/MCS/det/material` command (candidates: PLA, silicon, tungsten) sets material via `SetMaterial()`. All six geometry constructors and `PlaceWallSlab` use `fTargetMaterial` instead of hardcoded `fPLA`. Volume names unchanged — SteppingAction volume detection unaffected.

### Gap 3: Switch particle from electron to muon
- **Status: IMPLEMENTED (Session 3)**
- **Files:** `src/PrimaryGeneratorAction.cc`, `include/PrimaryGeneratorAction.hh`
- **Details:** Added `fParticleName` member (default `"e-"`), `SetParticle(const G4String& name)` method using `G4ParticleTable::FindParticle(name)`, and `/MCS/gun/particle` messenger command (candidates: `e-`, `mu-`). `GetParticleName()` getter available for analysis scripts. Electron remains the default.

### ~~Gap 4: Switch EM physics from option4 to option0 or option3~~
- **Status: IMPLEMENTED (Session 3)**
- **Files:** `MCSHighland.cc`, `src/PhysicsList.cc`, `include/PhysicsList.hh`
- **Details:** Added optional third command-line argument `emOption` (0, 3, or 4; default 4). `MCSHighland.cc` parses `argv[3]` and passes it to the `PhysicsList(G4int emOption)` constructor. The constructor switches on the value to register `G4EmStandardPhysics` (option0, Urban MCS), `G4EmStandardPhysics_option3`, or `G4EmStandardPhysics_option4` (EMZ/WentzelVI). Selected option is printed at startup.

### ~~Gap 5: Build stacked independent layers (N rectilinear layers with random phase offsets)~~
- **Status: IMPLEMENTED (Session 5)**
- **Files:** `src/DetectorConstruction.cc`, `include/DetectorConstruction.hh`, `src/DetectorMessenger.cc`, `include/DetectorMessenger.hh`
- **Details:** Added `fNLayers` member (default 1), `SetNLayers()` setter, and `ConstructStackedRectilinearLattice()` method. When `fNLayers > 1` and geometry is rectilinear, the sample thickness is divided into N equal z-slices with 0.05 mm air gaps between them. Each layer gets a random transverse phase offset (uniform within one cell period) via `CLHEP::RandFlat`. Wall volumes use the same "XWall"/"YWall" names so SteppingAction detection works unchanged. New macro command: `/MCS/det/nLayers` (integer, range 1–100).

Also accepts `--data-dir` and `--output-dir`. Prints cut configuration and per-file cut summary.
